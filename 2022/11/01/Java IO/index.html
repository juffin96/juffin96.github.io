Java IO

字符集

ASCII 字符集

美国信息交换标准代码（American Standard Code for Information Interchange），包括了数字、英文、符号。

ASCII 使用1个字节存储一个字符，一个字节是8位，总共可以表示128个字符信息。

GBK 码表

Windows系统默认的码表。兼容ASCII码表，也包含了几万个汉字，并支持繁体汉字以及部分日韩文字。

GBK是中国的码表，一个中文以两个字节的形式存储。

Unicode 码表

统一码、万国码，是计算机科学领域的一项业界字符编码标准。容纳了世界上大多数国家的所有常见文字和符号。Unicode会通过UTF-8，UTF-16，以及UTF-32的编码成二进制后在存储到计算机。

  • Unicode是万国码，以UTF-8编码后一个中文一般以三个字节的形式存储。
  • UTF-8也要兼容ASCII编码表。
  • 技术人员都应该使用UTF-8字符集编码。
  • 编码前和编码后的字符集要一致，否则会出现中文乱码。

Java 中编码与解码

String 编码

  • byte[] getBytes()

    使用平台的默认字符集将该String编码为一系列字节，将结果存储到新的字节数组中。

  • byte[] getBytes(String charsetName)

​ 使用指定的字符集将该String编码为一系列字节，将结果存储到新的字节数组中。

String 解码

  • String(byte[] bytes)

​ 通过使用平台的默认字符集解码指定的字节数组来构造新的String。

  • String(byte[] bytes, String charsetName)

​ 通过指定的字符集解码指定的字节数组来构造新的String。

Java 常用编码 StandardCharsets.UTF-8。

字节流

字节流分为输入流FileInputStream和输出流FileOutputStream。

文件字节输出流

FileOutputStream写数据：

  • public void write(int a)写入一个字节
  • public void write(byte[] buffer)写入一个字节数组
  • public void write(byte[], int pos, int len)写入一个字节数组的一部分

文件字节输入流

FileInputStream读数据：

  • pulic void read(int a)读取一个字节
  • public void read(byte[] buffer)读取一个字节数组
  • public void write(byte[], int pos, int len)读取一个字节的一部分

流的关闭与刷新

  • flush()刷新流，强制将缓冲流的缓冲区的数据一次写出，还可以继续写入数据
  • close()关闭流，释放资源，但是会在关闭之前先刷新流flush()。一旦关闭则不能写入数据。

字节输出流如何实现数据追加？

public FileOutputStream(String filepath, boolean append)使用重载的构造器传入append为true则创建可追加数据的字节输出流。

利用字节流复制文件

字符流

java.io.Reader和java.io.Writer是所有字符流的超类，它们和字节流的超类是平级的关系。

Reader和Writer是两个抽象类，里面规定了所有字符流都必须具备的读写字符的相关方法。

字符流最小读写单位是字符，但是底层实际还是读写字节，只是字符与字节的转换工作由字符流完成。

转换流

java.io.InputStreamReader和OutputStreamWriter是字符流中常用的一对实现类，实际开发中不直接操作它们，但是它们在流连接中是非常重要的一环。

实际开发中，还有更好用的字符高级流，但是其他的字符流都有一个共通点，不能直接连接在字节流上，而实际操作设备的流都是低级流同时也都是字节流，因此不能直接在流连接中串联起来，转换流是一对可以连接在字节流上的字符流，其他的高级字符流可以连接在转换流上，在流连接中起到“转换器”的作用。

缓冲流

高级流的一种，用来处理原始字节流或字符流，提高读写数据的性能。自带缓冲区（默认为8kb），可以提高原始字节流、字符流读写数据的性能。

  • 字节缓冲流

    • 字节缓冲输入流：public BufferedInputStream(InputStream is)包装一个低级的字节输入流`
    • 字节缓冲输出流：public BufferedOutputStream(OutputStream os)包装一个低级的字节输出流

  • 字符缓冲流

    • 字符缓冲输入流：public BufferedReader(Reader r)

    public String readLine()读取一行数据，如果读取没有完毕，无行可读返回null。

    • 字符缓冲输出流：public BufferedWriter(Writer w)

    public void newLine()换行操作。

    • 打印缓冲输出流：public PrintWriter(Writer w)

对象流

高级流的一种。对象流在流连接中的作用是进行对象的序列化与反序列化，其中对象输出流负责对象序列化，对象输入流负责对象反序列化。

所谓对象序列化，就是将写出的对象按照其结构转换为一组字节的过程。

对象反序列化。

实现序列化

需要进行序列化的类必须实现接口java.io.Serializable，实现该接口后最好主动定义序列化版本号这个常量，这样一来对象序列化是就不会根据类的结构生成一个版本号，而是使用该固定值，那么反序列化时，只要还原的对象和当前类的版本号一致就可以进行还原。

transient关键字可以修饰属性，用于在进行对象序列化时忽略不必要的属性，达到对象“瘦身”的目的。
